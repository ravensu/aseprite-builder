# aseprite-builder

GitHub Actions automation for building Aseprite with official-style source + Skia settings.

## What this workflow does

- Polls upstream Aseprite releases every 6 hours.
- Builds automatically when a new upstream source release appears.
- Produces:
  - `Aseprite-<tag>-windows-x64.zip`
  - `Aseprite-<tag>-macos-arm64.zip` (contains `Aseprite.app`)
  - `Aseprite-<tag>-macos-x64.zip` (contains `Aseprite.app`)
- Uses the Skia URL generated by Aseprite/LAF metadata (`laf/misc/skia-url.sh`), so Skia package selection follows upstream requirements.

## Manual trigger

Run workflow `Build and release Aseprite` from GitHub Actions:

- `aseprite_ref`: Optional upstream tag (e.g. `v1.3.17`), defaults to latest release with a Source zip asset.
- `force_rebuild`: Rebuild even when all assets already exist.

## Why this solves missing icon issues

- macOS artifacts package the generated `Aseprite.app` bundle directly, preserving app icon and bundle metadata.
- Windows artifacts package the full compiled `bin` runtime instead of pruning files aggressively.
